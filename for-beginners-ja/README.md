# Claude Code — 初心者向け

このディレクトリは、Claude Codeの4つの基本的な構成要素を実際に手を動かしながら学べる入門ガイドです:
**スラッシュコマンド (slash commands)**、**スキル (skills)**、**フック (hooks)**、そして**サブエージェント (subagents)**。

それぞれがコードレビューというテーマを通じて紹介されています。さらに、同じスキルの仕組みを
活用したオプションの計画ワークフロー（SDD）も用意されています。実際に試して、各ファイル内の
コメントを読めば、Claude Codeの仕組みについてしっかりとしたメンタルモデルが身につきます。

---

## 前提条件

- [Claude Codeをインストール済みであること](https://code.claude.com/docs)
- ターミナルでこのディレクトリに移動し、`claude` を実行してセッションを開始する
- レビューや説明を試すための小さなサンプルコードを用意しておくこと。手元に何もなければ、
  Claudeに作ってもらうのが手っ取り早いです。例えば三目並べ:
  ```
  tictactoe.js を作ってください
  ```
  またはブラックジャック:
  ```
  blackjack.rb を作ってください
  ```
  `/code-review` スキルはソースコードを対象に設計されているので、ドキュメント
  (例: README.md ファイル) ではなく、実際のプログラムを用意してください。
  その方が4つの構成要素の違いがはっきり体感できます。

---

## 4つの基本構成要素

### 1. スラッシュコマンド (Slash Command) — `/explain`
**ファイル:** `.claude/commands/explain.md`

スラッシュコマンドは最もシンプルな構成要素です。1つの `.md` ファイルが再利用可能なプロンプトになります。
スラッシュコマンドは**常に手動で実行されます** — Claudeが自動で呼び出すことはありません。

**試してみる:**
```
/explain README.md
```
または
```
/explain explain.mdファイル
```

**主な特徴:**
- 1つの `.md` ファイル(ディレクトリも補助ファイルもなし)
- 必ず手動で実行される
- シンプルで焦点が明確な、一発限りのタスクに最適

---

### 2. スキル (Skill) — `/code-review`
**ファイル:** `.claude/skills/code-review/SKILL.md` + `general-review-instructions.md`

スキルは外見上はスラッシュコマンドと似ていますが(`/code-review` と入力して呼び出せる)、内部的にはより
強力です。スキルは**ディレクトリ**内に存在し、補助ファイルを持つことができます。
さらに、Claudeが関連性を判断したときに**自動でトリガー**されることもあります。

**試してみる:**
```
/code-review
```

プロンプトが表示されたら、`auto`(自動)と `step-by-step`(ステップごと)の両方で試して、
パイプラインの動作の違いを確認してください。

**スラッシュコマンドとの主な違い:**
- ディレクトリ構造を持つ(単一ファイルではない)
- 補助的なコンテキストファイルを含められる(`general-review-instructions.md` を参照)
- Claudeによって自動でトリガーされる可能性がある(フロントマターの `disable-model-invocation` で制御)
- 複数ステップの複雑なワークフローを統括し、サブエージェントの起動も可能

---

### 3. サブエージェント (Subagent) — `security-reviewer`
**ファイル:** `.claude/agents/security-reviewer.md`

サブエージェントは、メインの会話から**隔離された**状態で動作する独立したClaudeインスタンスです。
独自の集中した指示を持ち、あなたとの会話履歴を引き継ぎません。

`security-reviewer` エージェントは `/code-review` スキルのステップ3で自動的に起動されます。
直接呼び出すこともできます:

```
security-reviewerエージェントを使って auth.js をレビューしてください
```

**なぜセキュリティレビューにサブエージェントを使うのか?**
- セキュリティレビューには会話のコンテキストは不要 — コードだけがあれば十分
- セキュリティに関する詳細な出力でメインのコンテキストウィンドウが圧迫されるのを防ぐ
- 専門化されたエージェントは、余計なことに気を取られず、1つのことを上手にこなす

---

### 4. フック (Hook) — PostToolUse
**ファイル:** `.claude/settings.json`

フックは、Claude Codeのイベントに応じて自動的に実行されるシェルコマンドです。
他の3つの構成要素とは異なり、フックは**プロンプト形式では書かれません** —
`settings.json` で設定するシェルコマンドです。

このプロジェクトのフックは、Claudeがファイルを書き込みまたは編集した後に発火し、
レビューを実行するよう促します。

**フックは手動でトリガーしません** — Claudeにファイルを編集してもらうだけで何が起きるか
観察してください:

```
hello.js というシンプルなファイルを作ってください
```

Claudeがファイルを書いた後、フックが自動で発火します。

**主な特徴:**
- スタンドアロンの `.md` ファイルではなく、`settings.json` で設定する
- ユーザーではなく、Claude Codeのイベント(例: ツール実行後)によってトリガーされる
- シェルコマンドを実行する — リマインダー、リンター、テストランナー、何でもできる

## 全体のワークフロー

4つの構成要素が一緒にどう連携するか:

```
1. Claudeにコードを書くように依頼する
        ↓
2. フックが発火 → /code-review の実行を促す
        ↓
3. /code-review (スキル) を実行
        ↓
4. スキルが質問: auto か step-by-step か?
        ↓
5. 一般レビューが実行される(general-review-instructions.md の基準を使用)
        ↓
6. セキュリティのサブエージェントが起動し、独立してレビュー
        ↓
7. 両方の修正が適用される。レビュー完了。
```

## オプション: コードを書く前に計画する（SDDワークフロー）

**SDD** は **Spec-Driven Development（仕様駆動開発）** の略で、コードを書く前に短い
仕様（何を作るのか、なぜ作るのか、どう作るのか）を書いておくという考え方です。重厚で形式的な
ものである必要はありません。各トピックについて数文書くだけでも、あなたとClaudeが共通の目標を
持つのに十分です。

軽量なSDDを体験できる2つの追加スキルが用意されています。`/code-review` で学んだスキルの
仕組みを、別の問題に適用したものです。

### 使い方

1. `/sdd-init` を実行 — Claudeが作りたいアプリについて5つの短い質問をします。
2. 回答をもとに、Claudeが4つの小さな計画ドキュメントを生成します:
   - `CLAUDE.md` — Claudeのためのプロジェクトレベルのガイダンス
   - `docs/requirements.md` — 何を作るのか、なぜ作るのか
   - `docs/design.md` — 技術設計（技術スタック、ファイル構成、コンポーネント）
   - `docs/tasks.md` — 順序付きの実装チェックリスト
3. (オプション) 4つのドキュメントを手動で編集して計画を調整できます。
4. `/sdd-build` を実行 — Claudeが4つのドキュメントを読み、タスクごとにアプリを実装します。
5. 生成されたコードに対して `/code-review` を実行 — すでに知っているスキルです。

### なぜSDDを使うと良いのか

- **質問を通じて自分のアイデアを明確にできます。** 初心者はぼんやりとしたイメージで
  スタートしがちです。5つの質問に答えるのに2分ほどですが、それによってあなたもClaudeも
  明確な目標を持てます。
- **コードを書く前に計画をレビュー・編集できます。** 設計がおかしいと思ったら
  `docs/design.md` を修正すれば良い — コードを書き直すよりずっと手軽です。
- **やり直しが簡単です。** 生成されたコードが思い通りでなくても、ドキュメントは残っています。
  コードを削除してドキュメントを修正し、`/sdd-build` を再実行するだけです。
- **ドキュメントがそのまま資料になります。** アプリ完成後も、1週間後にコードに戻ったときに
  「これ何だっけ？」とならずに済みます。

### SDDを使わなくても大丈夫

「三目並べを作って」とそのまま頼んでも問題ありません。SDDスキルは、何を作るかをもっと
コントロールしたいときのためのものです — 完全にオプションです。

## Tips & Tricks

すべてを網羅したリストではありません — 知っておくと便利な機能をいくつか紹介します。

### プロンプトのショートカット

- `! <command>` — チャットを離れずにシェルコマンドを実行する（`! npm test`、`! git status`）
- **画像を貼り付ける** — Claudeはスクリーンショットや図を読み取れます。貼り付けて質問してみてください

### スラッシュコマンド

- `/plan` — プランモードに入る。Claudeは変更を加える前にやることの概要を示す。Shift+Tabでプランモードに切り替えるのと同じ
- `/btw` — 会話履歴に残らないちょっとした質問ができる。回答は閉じるまで表示されるオーバーレイに表示され、閉じると消える。Claudeはセッション全体のコンテキストを参照できるが、新しいファイルを読んだりコマンドを実行したりはできない。Claudeがタスク実行中でも使える
- `/simplify` — 最近変更されたファイルをコードの再利用性・品質・効率性の観点でレビューし、修正する。生成されたコードが必要以上に複雑だと感じたときに便利
- `/cost` — 現在のセッションのトークン使用量の概要を表示する
- `/context` — コンテキストウィンドウの消費内訳を詳しく表示する（システムプロンプト、ツール、メッセージ、空き容量）
- `/usage` — プランの使用量制限と現在のレート制限状況を表示する
- `/compact` — 長い会話を圧縮してコンテキストを解放する（重要な情報は保持される）
- `/clear` — 会話を完全に消去し、コンテキストゼロでやり直す

### キーボードショートカット

- **Shift+Tab** — 3つのモードを切り替える: デフォルトモード（各アクションを承認）、編集自動承認モード（ファイル編集と `mkdir`、`touch`、`rmd`、`rmdir`、`mv` などのファイルシステムコマンドをワーキングディレクトリ内で自動承認）、プランモード（`/plan` と同じ — Claudeはやることの概要を示すが、アクションは取らない）
- **Ctrl+S** — 現在のプロンプトを一時保存する。もう一度 Ctrl+S を押すと復元され、次のプロンプトを送信した後にも自動で再表示される
- **Escape** — Claudeが間違った方向に進んでいるとき、タスクを中断する
- **Escape Escape** — `/rewind` と同じ。直前のターンを取り消し、Claudeが行ったファイル変更を元に戻す。または、選択したメッセージから会話を要約する

## ファイル構成

```
for-beginners-ja/
├── README.md                                   ← ここにいます
└── .claude/
    ├── settings.json                           ← フックの設定
    ├── commands/
    │   └── explain.md                          ← /explain スラッシュコマンド
    ├── skills/
    │   ├── code-review/
    │   │   ├── SKILL.md                        ← /code-review スキル(エントリーポイント)
    │   │   └── general-review-instructions.md  ← スキルの補助コンテキスト
    │   ├── sdd-init/
    │   │   ├── SKILL.md                        ← /sdd-init スキル(質問票)
    │   │   └── templates/                      ← /sdd-init が使うドキュメントテンプレート
    │   │       ├── CLAUDE.md.template
    │   │       ├── requirements.md.template
    │   │       ├── design.md.template
    │   │       └── tasks.md.template
    │   └── sdd-build/
    │       └── SKILL.md                        ← /sdd-build スキル(実装)
    └── agents/
        └── security-reviewer.md               ← セキュリティレビューのサブエージェント
```
