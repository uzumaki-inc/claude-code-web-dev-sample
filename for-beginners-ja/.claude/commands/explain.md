# /explain

ユーザーは以下について説明を求めています: **$ARGUMENTS**

ファイルパスが指定されている場合:
- そのファイルを読む
- 1〜2文でファイルの目的を説明する
- 主要なセクション、関数、またはロジックを説明する
- プロジェクト全体での位置づけがわかれば、それも説明する

関数名やクラス名が指定されている場合:
- コードベース内でそれを見つける
- 何をするのか、入力は何か、何を返すのかを説明する
- 自明でない動作や副作用があれば指摘する

$ARGUMENTS が空、または不明確な場合は、こう尋ねてください: 「何について説明しましょうか? ファイルパスまたは関数名を教えてください。」

説明はわかりやすくアクセスしやすいものにしてください。読み手はプログラミングは
理解しているが、この特定のコードには馴染みがないと想定してください。

すべての説明は**日本語**で行ってください。
